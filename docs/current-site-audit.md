# Current ZenCloud Site Audit

## Purpose

This audit captures the current public ZenCloud site state before migration from the existing hosted site to GitHub and Cloudflare.

Target repo: `ZenCloudAU/zencloud-advisory`
Target future domain: `https://www.zencloud.com.au/`

## Executive Finding

The current ZenCloud site contains useful advisory positioning, but it still presents as a legacy general advisory website rather than the front door for the full ZenCloud / StudioSix / Velocity Architecture ecosystem.

The new site should not become a complete archive of essays, research notes, book drafts, or every ecosystem asset. It should act as the commercial advisory front door and route visitors to the appropriate ecosystem destination.

## What Should Remain

Retain these themes:

- Enterprise architecture advisory
- Solution architecture and design authority
- Cloud strategy and platform modernisation
- Identity, security, and governance
- Technical leadership and program recovery
- AI-assisted architecture and delivery
- Founder credibility and enterprise experience
- Links to Velocity Architecture, StudioSix, Academy, Medium, Substack, and GitHub

## What Should Be Removed or Reworked

Remove or replace legacy/template content such as:

- Placeholder phone number text
- Placeholder custom text
- Generic inspirational quotes where they weaken commercial clarity
- Unnecessary street-level address exposure unless intentionally required
- Unclear navigation labels such as `VAFResearch`

Demote personal publishing links from the primary navigation. Medium and Substack should be available through Insights or Publishing, not as core commercial services.

## Current UX Issues

- Navigation does not explain the full ecosystem clearly.
- ZenCloud, StudioSix, Velocity, Academy, and tools are not yet presented as a coherent system.
- There is no clear commercial conversion path.
- Service packaging is not yet sharp enough for enterprise buyers.
- Research and essays risk overwhelming the advisory message.
- Profile content is useful but should be reframed from CV-style to trust/credibility proof.

## Current UX Strengths

- Existing advisory topics are relevant.
- Enterprise architecture and AI delivery positioning is strong.
- Founder profile has credibility.
- The phrase `Architecture at the speed of the decision` is valuable.
- The ecosystem already has real assets that can be routed from the site.

## Target Site Role

ZenCloud should be the advisory front door:

```text
ZenCloud advises.
StudioSix produces.
Velocity decides.
```

ZenCloud should route to:

- Velocity Architecture Framework
- Velocity Architecture Academy
- StudioSix
- EA Artefact Generator
- PMO Portal
- VAF-SA
- VSF Match
- Medium
- Substack
- GitHub

## Recommended Public Positioning

Primary statement:

```text
ZenCloud is an enterprise architecture and AI delivery advisory practice helping organisations turn complex technology decisions into governed, executable outcomes.
```

Support statement:

```text
ZenCloud helps leaders connect enterprise architecture, solution architecture, cloud, security, governance, and AI-assisted delivery into one decision-first operating model.
```

## Migration Principle

Do not lift-and-shift the old site.

Use the migration to clarify the brand, reduce noise, and make the ecosystem navigable.
